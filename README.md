# Action Board

Личный трекер действий, времени и прогресса к двум целям:

- получить оффер 1С-разработчика;
- прийти к форме 65-67 кг.

Приложение сделано на React + Vite + TypeScript + Tailwind CSS. Данные хранятся в Firebase Firestore, вход выполняется через Google.

## Запуск локально

```bash
npm install
npm run dev
```

После запуска откройте адрес, который покажет Vite, обычно `http://localhost:5173`.

## Подключение Firebase

1. Создайте проект в [Firebase Console](https://console.firebase.google.com/).
2. Откройте `Authentication`.
3. Включите провайдер `Google`.
4. Создайте `Firestore Database`.
5. Скопируйте `.env.example` в `.env`.
6. Добавьте настройки Firebase в `.env`.
7. Запустите проект локально через `npm run dev`.
8. Проверьте вход через Google.
9. Настройте Firestore Rules из файла `firestore.rules`.

Файл `.env` должен содержать:

```env
VITE_FIREBASE_API_KEY=
VITE_FIREBASE_AUTH_DOMAIN=
VITE_FIREBASE_PROJECT_ID=
VITE_FIREBASE_STORAGE_BUCKET=
VITE_FIREBASE_MESSAGING_SENDER_ID=
VITE_FIREBASE_APP_ID=
```

## Firestore структура

Используются коллекции внутри документа пользователя:

- `users/{uid}/days/{date}` - дневные метрики, очки, статус, артефакт и рефлексия.
- `users/{uid}/timeEntries/{entryId}` - остановленные таймеры и записи времени.
- `users/{uid}/goals/{goalId}` - редактируемые цели.
- `users/{uid}/settings/main` - зарезервировано под будущие настройки.

Минимальные Firestore Rules:

```js
match /users/{userId}/{document=**} {
  allow read, write: if request.auth != null && request.auth.uid == userId;
}
```

## Что умеет приложение

- Google Sign-In через Firebase Authentication.
- Экран `Сегодня`: таймеры по категориям, дневные метрики, очки, статус дня, артефакт и рефлексия.
- Экран `Неделя`: часы по категориям, часы 1С, практика, пет-проект, текущая работа, цвета дней и средние метрики.
- Экран `Цели`: базовые цели создаются автоматически и редактируются.
- Экран `История`: список дней с фильтрами по датам, цвету дня и категории.
- Экран `Экспорт`: выгрузка JSON или CSV за неделю, месяц, период или всю базу.
- Импорт JSON для восстановления или переноса данных.

## Экспорт отчета

1. Войдите через Google.
2. Откройте экран `Экспорт`.
3. Выберите формат `JSON` или `CSV`.
4. Нажмите `Экспорт за неделю`, `Экспорт за месяц`, `Экспорт за период` или `Экспорт всей базы`.

JSON-отчет удобнее загружать в ChatGPT: он содержит дневные метрики, тайм-записи, цели, артефакты, рефлексии и итоговую сводку за период.
