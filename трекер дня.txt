Создай полноценное веб-приложение “Action Board” — личный трекер действий, времени и прогресса к двум главным целям:

1. Смена профессии: стать 1С-разработчиком, получить оффер.
2. Форма тела: вес 65–67 кг, видимый пресс, стабильный режим питания и тренировок.

Важно: это не дневник настроения и не красивый трекер ради трекера. Это панель управления действиями. Главный вопрос приложения: “Что я сегодня сделал для оффера и тела?”

Технологический стек:

* React + Vite.
* TypeScript желательно, если проект уже на JS — можно JS, но код должен быть чистый и понятный.
* Tailwind CSS.
* Firebase:

  * Firebase Authentication через Google.
  * Firestore как база данных.
* Приложение должно работать в браузере на компьютере и телефоне.
* Должна быть адаптивная мобильная и десктопная версия.
* Данные должен видеть только авторизованный пользователь.
* Добавь README.md с понятной инструкцией подключения Firebase.

Главные экраны:

1. Экран авторизации

* Кнопка “Войти через Google”.

Профессия:

Прочее:

4. Система очков дня

Профессия:

* 1С 30+ минут: +1

* 1С 4+ часа: +4
* практика / финальная работа: +3

Тело:

* калории в коридоре: +2

Статус дня:

* 0–3 очка: красный день

* 12+ очков: боевой день

Важно: цвет дня должен отображаться визуально.

10. Firebase
    Добавь файлы:

* firebase.ts / firebase.js
* .env.example
* README.md

В .env.example должны быть:
VITE_FIREBASE_API_KEY=
VITE_FIREBASE_AUTH_DOMAIN=
VITE_FIREBASE_PROJECT_ID=
VITE_FIREBASE_STORAGE_BUCKET=
VITE_FIREBASE_MESSAGING_SENDER_ID=
VITE_FIREBASE_APP_ID=

В README.md напиши инструкцию:

1. Создать проект Firebase.
2. Включить Authentication.
3. Включить Google Sign-In.
4. Создать Firestore Database.
5. Добавить настройки Firebase в .env.
6. Запустить проект локально.
7. Проверить вход через Google.
8. Настроить Firestore Rules.

Firestore структура:
users/{uid}/days/{date}
users/{uid}/timeEntries/{entryId}
users/{uid}/goals/{goalId}
users/{uid}/settings/main

Security Rules:
Пользователь может читать и писать только свои данные:
match /users/{userId}/{document=**} {
allow read, write: if request.auth != null && request.auth.uid == userId;
}
